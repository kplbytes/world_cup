# FRONTEND_UI_RULES.md
# 2026 世界杯预测系统前端 UI 规则

本文件是前端页面、组件、布局和交互的长期约束。任何 AI 修改前端前必须先阅读本文件。

## 1. 前端定位

本系统不是数据大屏，也不是开发调试面板。它是用户每天查看世界杯预测的本地工作台。

用户打开页面后，第一眼必须能回答：

1. 今天有哪些比赛？
2. 系统预测谁更可能赢？
3. 风险高不高？
4. AI 和 Ensemble 是否已生成？
5. 我是否需要点击详情？
6. 赛后复盘是否有结果？

## 2. 四个主入口固定

顶层导航只保留：

1. 今日工作台
2. 比赛中心
3. 模型复盘
4. 冠军与赛程

不得恢复旧的多入口导航。

## 3. 今日工作台第一屏规则

今日工作台第一屏应优先展示：

1. 今日状态；
2. 下一步建议；
3. 已结束比赛复盘；
4. 未来 24 小时比赛列表。

不要让大标题、数据源状态、空白区域占据过多首屏空间。

### 3.1 动作区规则

首页动作区只保留统一的一组工作流入口，不要再在 Header 或其他位置复制一套同义按钮。

- 页面刷新只允许重新拉取展示数据，不允许自动触发 workflow
- `更新今日数据`、`同步赛果` 必须手动触发，且不显示 60 分钟冷却
- `运行 AI 预测` 必须手动触发，并显示 60 分钟冷却状态
- `一键更新全部` 必须明确会调用 AI API 产生费用
- 工作流运行中时，动作按钮和状态区都要展示同一份百分比进度

### 3.2 冠军与赛程 / 淘汰赛规则

- 淘汰赛页必须使用官方 Match 73-104 赛程和官方第三名组合表口径
- 未决出的席位显示为“待定”或 source ref，不要再展示“简化模拟”旧文案
- 已结束淘汰赛要自动把晋级结果写入下一轮显示
- 四个顶层导航在不同页面间位置必须保持一致，不能切页后漂移

## 4. 比赛卡片规则

比赛卡片不是调试面板。

每张卡片默认只展示：

1. 比赛时间；
2. 主队 vs 客队；
3. 综合推荐；
4. 预测比分；
5. 胜平负概率；
6. 风险等级；
7. 赛前快照状态；
8. 查看详情按钮。

Baseline / AI / Ensemble 的详细比较应放入详情抽屉，不应在卡片上堆太多格子。

## 5. 卡片布局规则

所有比赛卡片必须满足：

1. 不横向溢出；
2. 不互相遮挡；
3. 长队名不撑爆；
4. 概率条不跑出卡片；
5. 卡片高度基本一致；
6. 查看详情按钮位置固定；
7. 页面不得出现横向滚动条；
8. 右侧不得出现大面积无意义空白。

## 6. 比赛卡片推荐结构

推荐卡片结构：

顶部：

- 时间
- 状态
- 风险等级

中部：

- 主队
- VS
- 客队

核心结论：

- 综合推荐：主胜 / 平 / 客胜
- 预测比分：x : y

概率：

- 主胜 xx%
- 平局 xx%
- 客胜 xx%

底部：

- 赛前快照状态
- 查看详情

## 7. 详情抽屉规则

点击“查看详情”必须打开抽屉或弹窗，不允许在卡片内部展开长详情。

详情抽屉包含：

1. 综合结论；
2. Baseline / AI / Ensemble 对比；
3. Team Profile；
4. 风险解释；
5. 赛前快照状态；
6. 是否参与赛后评分；
7. AI 错误原因，如果有。

## 8. 中文化规则

所有用户可见球队名必须中文化。

不得出现：

- `Mexico vs South Africa`
- `Qatar vs Switzerland`
- `Brazil vs Morocco`

应显示：

- 墨西哥 vs 南非
- 卡塔尔 vs 瑞士
- 巴西 vs 摩洛哥

## 9. 时间规则

所有用户可见比赛时间默认显示北京时间。

今日比赛必须按北京时间自然日筛选。

首页主比赛列表固定展示从当前时刻起未来 24 小时内的未完赛比赛，标题必须写“未来 24 小时比赛（北京时间）”。已完赛比赛不得混入该列表，应进入复盘区并显示比分与评分口径。

如果显示未来 48 小时，标题必须写“未来 48 小时比赛”，不能写“今日比赛”。

## 10. 风险显示规则

风险标签不能滥用。

如果 AI 未运行，不要全部显示高风险，应显示：

`AI 未运行，风险待确认`

风险等级应分为：

- 低风险
- 中风险
- 高风险
- 待确认

## 11. Mock 数据提示

如果 Team Profile 使用 `seed_mock_v1`，前端必须显示：

`数据来源：seed_mock_v1，仅用于功能验证，不代表真实历史表现。`

## 12. 前端验收必须截图

任何前端任务完成后，必须提供或说明以下截图检查结果：

1. 今日工作台 `1440px`；
2. 今日工作台 `1280px`；
3. 今日工作台 `768px`；
4. 今日工作台 `390px`；
5. 比赛详情抽屉；
6. 模型复盘；
7. 比赛中心。

必须确认：

1. 无横向滚动条；
2. 卡片不溢出；
3. 右侧无大面积无意义空白；
4. 中文队名正常；
5. 时间为北京时间；
6. 查看详情可打开；
7. 用户第一眼能看懂推荐结果。

## 13. 禁止行为

禁止：

1. 只跑 typecheck 就说前端完成；
2. 只看代码不看页面；
3. 卡片内部展开长详情；
4. 把所有模型细节堆在首页卡片；
5. 恢复 9 个顶层导航；
6. 用英文队名；
7. 用 UTC 日期筛今日比赛；
8. 用调试字段冒充用户文案；
9. 页面右侧大面积空白；
10. 未截图验收就宣称 UI 修复完成。
