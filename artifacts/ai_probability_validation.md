# AI 概率校验报告

## 修复内容

### AI 输出概率强校验

**问题：** AI 模型返回的 `home_win`、`draw`、`away_win` 概率未做任何校验，可能存在：
- 值超出 [0, 1] 范围（如 50/30/20）
- 总和偏差过大（如 0.2/0.2/0.2 = 0.6）
- 负值

**修复：** 在 `service.py` 的 `run_ai_prediction` 函数中，解析 AI 响应后添加概率校验：

1. **范围检查**：每个概率值必须在 [0, 1] 范围内
2. **总和检查**：概率总和必须在 [0.80, 1.20] 范围内
3. **归一化**：总和接近 1.0（误差 ≤ 0.02）时自动归一化
4. **拒绝**：超出范围时设置 `error_code = "invalid_probabilities"`，raw_response 仍保存

**校验规则：**
- 0.5 / 0.3 / 0.2 → 通过（总和 1.0）
- 0.49 / 0.30 / 0.20 → 归一化（总和 0.99）
- 50 / 30 / 20 → 拒绝（超出 [0,1]）
- 0.2 / 0.2 / 0.2 → 拒绝（总和 0.6 < 0.80）
- -0.1 / 0.6 / 0.5 → 拒绝（负值）

**修改文件：** `backend/app/ai/service.py`

**新增测试：** `tests/test_ai_improvements.py`（6 个概率校验测试）

## 剩余风险

- 校验阈值（0.80/1.20）为硬编码，未来可能需要根据实际 AI 输出质量调整。
- 归一化阈值 0.02 可能对某些模型过严，但宁可拒绝也不接受低质量输出。

## 对准确率链路的影响

**正面影响**：非法概率不再进入 ensemble 计算，避免了 ensemble 结果被污染。之前如果 AI 返回 50/30/20 这样的概率，ensemble 会计算出完全错误的结果。现在这类输出会被标记为 `invalid_probabilities`，不参与 ensemble。

**不影响**：合法的 AI 预测仍然正常进入 ensemble 和评分。
